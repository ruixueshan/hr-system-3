## 1. 项目概述

### 1.1 系统定位

展瑞人力资源管理系统 3.0，面向劳务派遣行业的内部管理平台，覆盖候选人招募、面试管理、员工入职、工时考勤、薪资核算、保险管理、财务管理、推荐提成全流程。

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 小程序端 | 原生小程序 + 云开发 SDK |
| Web 管理端 | Vue 3 + Vite + TypeScript + Element Plus |
| 后端 | 腾讯云 CloudBase 云函数（Serverless） |
| 数据库 | CloudBase NoSQL（MongoDB 兼容） |
| 第三方集成 | 云工保（保险 API）、企业微信（通知推送） |

### 1.3 角色权限体系

| 角色 | 平台 | 范围 |
|------|------|------|
| 总经理（gm） | Web | 全部权限 |
| 副经理（deputy） | Web | 全部权限 |
| HR | Web + 小程序 | 候选人、岗位、面试、入职、工时 |
| 外协（external） | 小程序 | 仅推荐，无管理权限 |
| 财务（finance） | Web | 薪资、提成、配置 |
| 候选人/员工 | 小程序 | 仅查看自身数据 |

## 2. 系统架构

### 2.1 整体架构

```
小程序 / Web 前端
    │
    ▼
云函数（Serverless）
    ├── auth-*           ← 登录认证
    ├── qrcode           ← 二维码报名/签到
    ├── applications     ← 报名管理
    ├── interviews       ← 面试管理
    ├── employees        ← 员工入职/管理
    ├── worktime         ← 工时考勤
    ├── salaries-v2      ← 薪资计算
    ├── salary-engine-v2 ← 薪资引擎（核心计算）
    ├── insurance-yungongbao ← 云工保保险
    ├── finance          ← 财务核算
    ├── stats-report     ← 报表推送
    └── notification     ← 通知/企业微信
    │
    ▼
CloudBase NoSQL（数据库）
    集合清单详见第4章
```

### 2.2 薪酬引擎独立服务（salary-engine）

**原则**：所有薪资、提成、利润计算必须通过统一的 `salary-engine-v2` 云函数入口，禁止前端或其他云函数直接操作薪资相关集合。

**函数列表**：
- `calculate-salary` — 薪资计算
- `calculate-bonus` — 提成计算
- `calculate-profit` — 利润核算
- `get-salary-detail` — 薪资明细
- `get-bonus-detail` — 提成明细
- `export-salary` — 薪资导出

## 3. 核心数据模型

### 3.1 三表边界原则（最高优先级）

```
users               = 账号（登录、权限、员工绑定）
employees           = 员工自然人主档（姓名、身份证、手机号、银行卡）
employee_companies  = 员工与企业的一段雇佣关系（企业、岗位、入离职、推荐人、工价）
```

**核心规则：**
- `employees` 一人一档，不绑定企业
- `employee_companies` 一人可多条（多次入职不同企业或同一企业）
- `employee_companies` 写入前必须先有对应的 `employees` 记录
- 姓名、身份证号、手机号的主数据 **只放在 `employees`**
- 企业、岗位、推荐人、工价 **只放在 `employee_companies`**

### 3.2 三表关系图

```
users ──employee_id──→ employees ──employee_id──→ employee_companies ──company_id──→ companies
                                                              │──job_id───────→ jobs
                                                              │──rate_plan_id──→ rate_plans
                                                              │──referrer_id───→ users
```

### 3.3 `users` — 统一账号表

**定位**：小程序和 Web 端的登录账号，不保存员工实名主数据。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_id` | string | 是 | 用户账号 ID |
| `openid` | string | 小程序必填 | 微信小程序登录标识 |
| `unionid` | string | 否 | 微信开放平台统一标识 |
| `phone` | string | 否 | **登录手机号**，不是员工手机号主档 |
| `name` | string | 否 | 昵称或展示名 |
| `real_name_snapshot` | string | 否 | 实名快照（兼容字段） |
| `avatar` | string | 否 | 头像 |
| `user_type` | string | 是 | `candidate` / `employee` / `admin` |
| `role` | string | 否 | 后台角色：`gm` / `deputy` / `hr` / `external` / `finance` |
| `status` | string | 是 | `normal` / `disabled` |
| `employee_id` | string | 否 | **绑定的员工主档 ID**（核心关联） |
| `employee_no_snapshot` | string | 否 | 员工编号快照 |
| `last_login` | date | 否 | 最近登录时间 |
| `created_at` | date | 是 | 创建时间 |
| `updated_at` | date | 是 | 更新时间 |

**禁止新增写入的字段：**
- ❌ `id_card` — 只作为兼容读取，不再新增写入
- ❌ `company_id` / `company_name`
- ❌ `job_id` / `job_name`
- ❌ `join_date` / `leave_date`
- ❌ `referrer_id` / `referrer_name` / `source_referrer_id` / `source_referrer_name`

### 3.4 `employees` — 员工自然人主档

**定位**：一人一档，不绑定任何某次入职关系。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_id` | string | 是 | 员工主档 ID |
| `user_id` | string | 否 | 绑定的用户账号 ID |
| `employee_no` | string | 是 | **员工编号，逻辑唯一** |
| `name` | string | 是 | **员工真实姓名**（主数据） |
| `phone` | string | 否 | **员工主手机号**（主数据） |
| `id_card` | string | 否 | **员工身份证号**（主数据） |
| `gender` | number | 否 | 性别 |
| `birth_date` | date | 否 | 出生日期 |
| `bank_name` | string | 否 | 开户行 |
| `bank_account` | string | 否 | 银行卡号 |
| `bank_account_name` | string | 否 | 开户名 |
| `bank_card_last4` | string | 否 | 银行卡后四位 |
| `emergency_contact` | string | 否 | 紧急联系人 |
| `emergency_phone` | string | 否 | 紧急联系人电话 |
| `is_blacklisted` | boolean | 否 | 是否黑名单 |
| `blacklist_reason` | string | 否 | 黑名单原因 |
| `merged_into_employee_id` | string | 否 | 合并后指向保留员工 |
| `merge_reason` | string | 否 | 合并原因 |
| `status_cache` | string | 否 | 由在职关系聚合的状态缓存 |
| `created_at` | date | 是 | 创建时间 |
| `updated_at` | date | 是 | 更新时间 |

**禁止新增写入的字段：**
- ❌ `company_id` / `company_name`
- ❌ `job_id` / `job_name`
- ❌ `join_date` / `leave_date` / `departure_date` / `departure_reason`
- ❌ `referrer_id` / `referrer_name` / `source_referrer_id` / `source_referrer_name` / `recommender_id` / `recommender_name`
- ❌ `hourly_rate` / `salary_type` / `settlement_mode` / `rate_plan_id`
- ❌ `contract_status` / `contract_no` / `contract_sequence` / `contract_signed_at` / `contract_start` / `contract_end` / `contract_type`

**约束：**
- `employee_no` 逻辑唯一
- `user_id` 非空时逻辑唯一
- `id_card` 非空时逻辑唯一（清理重复后启用）
- 被合并档案不物理删除，只设 `merged_into_employee_id`

### 3.5 `employee_companies` — 员工-企业在职关系表

**定位**：员工在某企业的一段雇佣关系。一人可多条。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_id` | string | 是 | 关系主键 |
| `employee_id` | string | 是 | → `employees._id` |
| `company_id` | string | 是 | → `companies._id` |
| `job_id` | string | 否 | → `jobs._id` |
| `rate_plan_id` | string | 否 | 工价方案 |
| `salary_type` | string | 否 | `monthly` / `daily` / `hourly` / `piece` |
| `settlement_mode` | string | 否 | `monthly` / `daily` |
| `hourly_rate` | number | 否 | 兼容字段，优先从 `rate_plan` 取 |
| `join_date` | date | 是 | **本段关系入职日期** |
| `leave_date` | date | 否 | **本段关系离职日期** |
| `status` | string | 是 | `active` / `left` / `pending` / `suspended` / `cancelled` |
| `referrer_id` | string | 否 | **推荐人 ID**（核心） |
| `referrer_type` | string | 否 | `user` / `employee` / `external` |
| `source_channel` | string | 否 | 来源渠道 |
| `company_name_snapshot` | string | 否 | 企业名称快照 |
| `job_name_snapshot` | string | 否 | 岗位名称快照 |
| `referrer_name_snapshot` | string | 否 | 推荐人名称快照 |
| `rate_plan_name_snapshot` | string | 否 | 工价方案名称快照 |
| `contract_status` | string | 是 | `unsigned` / `signed` |
| `contract_no` | string | 否 | 合同编号 |
| `contract_sequence` | number | 否 | 合同流水号 |
| `contract_signed_at` | date | 否 | 签约时间 |
| `contract_start` | date | 否 | 合同开始 |
| `contract_end` | date | 否 | 合同结束 |
| `contract_type` | string | 否 | 合同类型 |
| `insurance_policy_id` | string | 否 | 保险方案 ID |
| `insurance_mapping_id` | string | 否 | 保险映射 ID |
| `insurance_status` | string | 否 | 保险状态 |
| `created_by` | string | 否 | 创建人 |
| `created_at` | date | 是 | 创建时间 |
| `updated_at` | date | 是 | 更新时间 |

**状态枚举：**

| 状态 | 含义 | 规则 |
|------|------|------|
| `active` | 当前有效在职 | `leave_date` 为空或 ≥ 今天 |
| `left` | 已离职 | 必须有 `leave_date` |
| `pending` | 待生效 | 入职日期晚于今天 |
| `suspended` | 暂停 | 临时停用，不等于离职 |
| `cancelled` | 取消 | 不参与工时/薪资 |

**推荐字段映射（兼容期统一）：**

| 旧字段 | 新字段 |
|--------|--------|
| `source_referrer_id` | → `referrer_id` |
| `recommender_id` | → `referrer_id` |
| `referrer_name` | → `referrer_name_snapshot` |
| `source_referrer_name` | → `referrer_name_snapshot` |
| `recommender_name` | → `referrer_name_snapshot` |

**约束：**
- `employee_id` 必须存在于 `employees`
- `company_id` 必须存在于 `companies`
- 同一 `employee_id + company_id` 只能有一条 `active`
- 合同编号格式：`HT-{企业编码}-{5位流水}`

### 3.6 数据归属总表

| 数据 | 主归属 | 兼容读取 | 禁止写入 |
|------|--------|----------|----------|
| 登录 openid | `users` | — | 其他表不存 |
| 登录手机号 | `users.phone` | — | — |
| 员工姓名 | `employees.name` | `users.real_name_snapshot` | `users` 不再主写 |

| 企业归属 | `employee_companies.company_id` | `employees.company_id` 历史兼容 | `employees` 不再主写 |
| 岗位归属 | `employee_companies.job_id` | `employees.job_id` 历史兼容 | `employees` 不再主写 |
| 入职日期 | `employee_companies.join_date` | `employees.join_date` 历史兼容 | `employees` 不再主写 |
| 离职日期 | `employee_companies.leave_date` | `employees.leave_date` 历史兼容 | `employees` 不再主写 |
| 推荐人 | `employee_companies.referrer_id` | `users.referrer_id` 历史兼容 | `users` 不再主写 |
| 工价方案 | `employee_companies.rate_plan_id` | `employees.rate_plan_id` 历史兼容 | `employees` 不再主写 |
| 展示名称 | 关联查询生成 | `*_snapshot` 快照 | 不散落 `*_name` 字段 |

## 5. 数据写入边界

### 5.1 候选人注册

```
写 users：
  openid / phone / name
  user_type = candidate
  status = normal

禁止写 users：
  ❌ id_card（候选人阶段采集的身份证只放 applications）
  ❌ referrer_id / referrer_name
```

### 5.2 扫码报名（job-apply）

**输入**：`code`、`real_name`、`phone`、`id_card`

```
Step 1：查找或创建 users
  已存在 → 更新 name / phone（账号信息）
  不存在 → 新建 users（name, phone, role:candidate）
  ❌ 不写 id_card 到 users
  ❌ 不写 referrer_id/name 到 users

Step 2：调用 applications 云函数
  写入 applications：
    applicant_name / applicant_phone / applicant_id_card
    recommender_id / recommender_name
    status = pending

Step 3：自动创建面试记录
  写入 interviews：
    application_id / user_id
    candidate_name / phone
    interview_date / result = pending

Step 4：更新 qr_codes 使用次数
```

### 5.3 面试签到（interview-checkin）

**输入**：`code`、`real_name`、`phone`

```
Step 1：查或建 users（同报名流程）
Step 2：查找面试记录
Step 3：记录签到
Step 4：更新 interviews 状态
```

### 5.4 办理入职

**输入**：`user_id`、`job_id`、`join_date`、`referrer_id`

```
Step 1：查找可复用 employees
  优先级：id_card → user_id → phone + name
  找到 → 复用
  未找到 → 创建 employees（name, phone, id_card, employee_no）

Step 2：创建 employee_companies
  employee_id / company_id / job_id
  join_date / status = active
  referrer_id
  hourly_rate / salary_type / settlement_mode

Step 3：回写 users.employee_id
```

### 5.5 员工离职

```
只更新 employee_companies：
  leave_date
  status = left
  updated_at

employees.status_cache 只能由聚合任务更新，不由离职表单直接写
```

### 5.6 员工资料编辑

```
已绑定员工 → 写 employees：
  name / phone / id_card
  bank_* / emergency_*
  ❌ 不写 company_id / job_id / referrer_id 等关系字段

未绑定员工 → 可暂存 users（候选人快照）
```

## 6. 数据读取边界

### 6.1 登录后获取身份

```
输入：openid 或登录手机号

1. 查 users
2. 如果 users.employee_id 存在 → 查 employees（获取实名信息）
3. 查 employee_companies 当前有效关系（获取企业/岗位信息）

返回：
  - 账号信息 ← users
  - 员工实名信息 ← employees.name / phone / id_card
  - 企业岗位信息 ← employee_companies + companies / jobs
```

### 6.2 员工列表

```
主表：employees
关联：
  - 当前关系：employee_companies WHERE status = 'active'
  - 企业名称：companies.name
  - 岗位名称：jobs.position

禁止：
  ❌ 直接用 employees.company_id 过滤企业
  ❌ 直接用 employees.job_name 展示岗位
```

### 6.3 员工详情

```
返回三块：
  profile           ← employees 主档
  current_relations ← employee_companies WHERE status = 'active'
  history_relations ← employee_companies ALL
```

### 6.4 工时/薪资/保险

```
查询条件：employee_id + company_id + work_date
解析规则：join_date <= work_date
         leave_date 为空或 leave_date >= work_date
         status != 'cancelled'

禁止回退：
  ❌ employees.company_id
  ❌ employees.job_id
  ❌ employees.hourly_rate
```

## 7. 核心业务流

### 7.1 完整链路图

```
候选人扫码报名
    │
    ├── 创建/更新 users（账号）
    ├── 创建 applications（报名记录，含 id_card / recommender）
    └── 创建 interviews（面试记录，自动同步）
          │
          面试通过
          │
          ├── 创建 employees（主档：name, phone, id_card, employee_no）
          ├── 创建 employee_companies（关系：company, job, referrer）
          └── 回写 users.employee_id
                │
                每日打卡
                │
                ├── 写 worktimes（工时记录）
                ├── 汇总 worktime_monthly_summaries
                └── 计算 salaries（薪资）
                      │
                      离职
                      │
                      └── employee_companies.leave_date + status = left
```

### 7.2 定时任务

| 函数 | 时间 | 说明 |
|------|------|------|
| `stats-report` 日报 | 08:00 / 18:00 | 推送今日报名/面试/本月入职/当前在职 |
| `stats-report` 月报 | 周六18:00 | 推送当月报名/本月入职/累计在职 |
| `employees` 离职同步 | 00:05 每日 | 同步 employee_companies 离职状态 |
| `candidates` 公海回收 | 00:05 每日 | 回收过期候选人 |

### 7.3 报表统计规则

#### 日报数据源

| 指标 | 数据源 | 字段条件 |
|------|--------|---------|
| 今日报名 | `applications` | `apply_time >= 今天0点`，按 `recommender_name` 分组 |
| 今日面试 | `interviews` | `interview_date = 今天`，反查 `applications` 拿推荐人 |
| 本月入职 | `employee_companies` | `join_date >= 本月1日`，只读 `referrer_id` |
| 当前在职 | `employee_companies` | `status = active`，`leave_date` 为空或晚于今天 |

**禁止：** 从 `employees` 回退推荐人信息。

#### 月报数据源

| 指标 | 数据源 | 字段条件 |
|------|--------|---------|
| 当月报名 | `applications` | `apply_time >= 本月1日` |
| 本月入职 | `employee_companies` | `join_date >= 本月1日` |
| 累计在职 | `employee_companies` | `status = active` |

**推荐人白名单：** 通过 `system_config` 的 `report_recommender_list` 配置控制。

## 11. 索引策略

### 11.1 必加复合索引

#### worktimes
- `{ employee_id, company_id, work_date }` — 员工+企业+月份查询
- `{ company_id, work_date }` — 企业维度统计
- `{ employee_id, status }` — 员工待审核工时

#### salaries
- `{ employee_id, year, month }` — 员工历史薪资
- `{ company_id, year, month }` — 企业薪资核算

#### employee_companies
- `{ employee_id, status }` — 查询员工当前企业（核心）
- `{ company_id }` — 企业员工列表
- `{ referrer_id, join_date }` — 按推荐人统计入职

#### users
- `{ openid }` — 登录查询
- `{ phone }` — 手机号查询
- `{ employee_id }` — 员工绑定查询

#### employees
- `{ employee_no }` — 员工编号查询
- `{ user_id }` — 用户绑定查询
- `{ id_card }` — 身份证查询
- `{ merged_into_employee_id }` — 合并档案查询

#### applications
- `{ user_id }` — 我的报名
- `{ job_id }` — 岗位报名统计
- `{ apply_time }` — 日报统计

#### interviews
- `{ company_id, interview_date }` — 面试日历
- `{ application_id }` — 获取面试记录
