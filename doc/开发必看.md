# hr-system-3.0 Web 开发必看

## 1. 技术栈与目录
- 前端：Vue 3 + TypeScript + Element Plus + Vite
- 目录：`web/src`
  - `views/`：页面
  - `api/`：CloudBase 直接访问封装（见 `api/modules/*`）
  - `utils/`：工具
  - 路由：`src/router/index.ts`
  - 布局与菜单：`src/views/Layout/*`

## 2. 云资源与约定
- 数据库：CloudBase，前端直连；开发期集合权限建议“所有用户可读，创建者可读写”。
- 常用集合：`users`、`companies`、`jobs`、`employees`、`rate_plans`、`commission_plans`、`worktimes`、`salaries`、`advances`、`operation_logs`、`roles`、`candidates`/`applications`/`interviews`。
- 认证：`api/cloud.ts`，默认匿名；登录在 `api/modules/auth.ts`。

## 3. 核心页面/功能
- 企业 `/companies`：增删改查，状态筛选。
- 岗位 `/jobs`：发布/编辑/停止/启动招聘，绑定工价方案。
- 候选人 `/candidates`：来源/状态筛选，历史报名，安排面试。
- 面试 `/interviews`：批量通过/未通过/未参加/入职，结果筛选。
- 员工 `/employees`：入职登记/导入/导出，推荐人下拉（取 `users` 中可作为推荐人的账号，排除 candidate/employee），工价方案绑定。
- 工价方案 `/rate-plans`：CRUD，软删除。
- 提成方案 `/commission-plans`：模板/个人，提成方式=时长*单价 | 出勤比例*月额。
- 薪资 `/salary`：Tab 分“月结”“日结”，本地计算演示（未落库）。
- 工时 `/worktime`：查询、导入、审批。
- 预支 `/advance`：申请（金额/次数上限来自系统配置）、审批、打款。
- 系统设置 `/settings`：
  - 用户管理：新增用户（默认密码 `sqzr1234`）、同步 HR。
  - 角色权限：CRUD，多选权限。
  - 系统参数：预支规则（比例/单次上限/月次数/跨月），本地存储。
  - 操作日志：查询、导出，读 `operation_logs`。

## 4. API 模块要点（`src/api/modules/`）
- 统一 `getDatabase()`；日志用 `logAction(action, resource, details)`（写 `operation_logs`）。
- 主要模块：`companies.ts`、`jobs.ts`、`employees.ts`、`ratePlans.ts`、`commissionPlans.ts`、`advances.ts`、`users.ts`、`roles.ts`、`operationLogs.ts`。
- 新增/更新注意 `_id/id` 兼容，软删除用 `status/is_deleted`。

## 5. 权限/菜单
- 路由 `meta.permission` 与角色权限对应，如 `advance:manage`、`salary:manage`。
- 侧边栏：`views/Layout/Sidebar/Index.vue`。

## 6. 预支规则
- 存本地 `hr3_system_config`（后续可迁移后端）
  - `advance_limit_percent`、`advance_max_amount`、`advance_monthly_times`、`advance_allow_cross_month`
- 预支申请校验单次金额 & 月次数上限。

## 7. 薪资拆分
- 单页 Tab：月结、日结
  - 月结：按月聚合工时，状态流转演示。
  - 日结：按日工时*时薪演示。
- 结果未落库；落库可扩展 `salariesApi`。

## 8. 待办与风险
- 预支规则仅本地存储，需迁移云端配置。
- 薪资计算未扣预支/税，未写入 DB。
- 日志在前端调用后写 DB，安全性可提升到后端。
- 权限目前主要前端控制，后端未强校验。
- 提成方案尚未集成到薪资计算。
- 薪资/日结导出仍为占位。

## 9. 开发指引
- 新增字段：同步修改 `api/types.ts`、对应 API 模块、页面表单/列表。
- 新操作记得 `logAction`。
- 下拉控件建议宽度 220–240px 避免截断。

## 10. 构建部署
- 开发：`npm run dev`
- 构建：`npm run build`
- 部署（CloudBase Hosting）：`npm run deploy`

## 11. 快速参考
- 默认新用户密码：`sqzr1234`
- 日志集合：`operation_logs`
- 推荐人数据源：`users` 集合 `role=hr`
- 提成结算主表：`recruitment_bonus_batches`，明细表：`recruitment_bonuses`
- 提成计算/审核/发放统一走 `salary-engine`，前端只读批次和明细
- Web 提成页 `/bonus` 已包含“规则管理”页签，规则层级支持 推荐人+企业 / 推荐人 / 企业 / 全局默认
- 正式规则可通过 `bonus-config` 的 `generate-formal-rules` 批量生成；当前环境已用该动作替换初始化兜底规则
- 软删字段：`status='deleted'` + `is_deleted=true`（方案类）

## 12. 优先级建议（后续开发）
1) 将系统配置迁移到云端配置集合，并在前端读取。
2) 薪资计算落库，集成预支扣减/提成方案/税费。
3) 完善导出（薪资、日结、工时）。
4) 后端统一写操作日志，补权限校验。
