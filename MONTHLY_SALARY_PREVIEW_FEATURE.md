# 月结发薪预览与手工调整功能

## 📋 功能概述

新增月结发薪的预览和手工调整功能，改变原来的"计算→直接保存"流程为"计算→预览→手工调整→确认保存"。这样用户可以在保存前对薪资数据进行验证和调整。

## 🔄 工作流程

### 原流程（已改进）
```
计算薪资 → 直接保存到数据库 → 显示结果
```

### 新流程（改进后）
```
计算薪资 → 加载预览数据 → 弹出预览对话框 → 手工调整 → 确认保存 → 更新数据库
```

## ✨ 功能特性

### 1. 预览对话框
- **位置**：计算完成后自动弹出
- **内容**：展示已计算的月结薪资列表
- **统计信息**：实时显示
  - 总应发工资
  - 手工调整总额
  - 总实发工资
  - 预计支出

### 2. 手工调整功能
| 功能 | 说明 |
|------|------|
| 手工调整金额 | 支持正数（奖励）或负数（扣减） |
| 调整备注 | 记录调整原因，便于审计 |
| 实时计算 | 实发 = 基础薪资 + 手工调整 |
| 汇总统计 | 自动统计所有调整的总数 |

### 3. 确认流程
- **预览阶段**：用户可以修改每条记录的手工调整金额和备注
- **确认按钮**：点击"确认提交审核"后执行以下操作：
  1. 将所有薪资记录状态更新为"已审核"（approved）
  2. 保存手工调整金额和备注
  3. 保存到数据库
  4. 自动关闭预览对话框
  5. 刷新主列表显示最新数据

- **取消按钮**：
  1. 放弃所有修改
  2. 回到主列表
  3. 需要重新计算才能再次预览

## 📁 实现细节

### 后端（云函数）- `/cloudfunctions/salaries-v2/index.js`

#### 新增接口：`monthly-preview`
```javascript
case 'monthly-preview': return await handleMonthlyPreview(event);
```

**功能**：查询已计算的月结薪资数据（不修改数据库）

**参数**：
```json
{
  "company_id": "企业ID",
  "year": 2026,
  "month": 4
}
```

**返回**：
```json
{
  "code": 0,
  "message": "成功获取 N 条月结薪资预览数据",
  "data": [
    {
      "_id": "记录ID",
      "employee_id": "员工ID",
      "employee_no": "工号",
      "employee_name": "员工名",
      "gross_pay": 5000,
      "insurance_deduct": 100,
      "tax": 500,
      "net_pay": 4400,
      "manual_adjust": 0,
      "adjust_remark": "",
      "status": "calculated"
    }
  ]
}
```

**查询条件**：
- settlement_mode = 'monthly'
- status = 'calculated'
- 按 employee_no 排序

## 🔐 权限与状态

### 账户权限
- HR、财务、总经理、副总经理可访问该功能

### 薪资状态流转

```
未计算
   ↓
[点击计算薪资]
   ↓
已计算 (calculated) ← 此时显示预览
   ↓
[预览对话框中点击确认]
   ↓
已审核 (approved) ← 进入发薪流程
   ↓
[标记发放]
   ↓
已发放 (paid)
```

## 💾 数据库影响

### salaries 表字段
| 字段 | 变化 |
|------|------|
| `manual_adjust` | 从预览对话框中保存 |
| `adjust_remark` | 从预览对话框中保存 |
| `status` | calculated → approved |
| `updated_at` | 更新为当前时间 |

### 新增字段说明
- `manual_adjust`：浮点数，可正可负，默认 0
- `adjust_remark`：字符串，最长 500 字符

## 📊 使用场景

### 场景1：正常发薪
1. HR 选择企业和月份
2. 确认工时汇总已全部审核
3. 点击"计算薪资"
4. 预览对话框显示，检查数据无误
5. 点击"确认提交审核"
6. 薪资保存并进入发薪流程

### 场景2：奖励调整
1. 同上计算和预览
2. 在预览中找到张三的记录
3. 手工调整：+500（年终奖）
4. 备注：年终奖 500 元
5. 点击"确认提交审核"
6. 张三的实发 = 应发 + 500

### 场景3：扣减调整
1. 同上计算和预览
2. 在预览中找到李四的记录
3. 手工调整：-100（旷工扣款）
4. 备注：旷工一天，扣 100 元
5. 点击"确认提交审核"
6. 李四的实发 = 应发 - 100

### 场景4：放弃修改
1. 进行了手工调整
2. 发现有错误，需要重新调整
3. 点击"取消"
4. 确认放弃修改
5. 回到主列表
6. 重新点击"计算薪资"重新预览

## 🔄 与日结发薪的对应

此功能与日结发薪的"日结薪资预览"功能保持一致：
- 日结预览接口：`daily-preview`
- 月结预览接口：`monthly-preview`（新增）
- 都支持手工调整和备注
- 都需要用户确认后才保存
